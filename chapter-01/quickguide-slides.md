# 🤔 一言で言うと

## REST API
**レストランのメニュー（紙）**

## MCP  
**スマートなウェイター（AI対応）**

---

# 📖 REST API = 紙のメニュー

## 使い方

1. 紙のメニューを見る
2. 注文を伝える
3. ウェイターが持ってくる

---

# REST APIの特徴

### ✅ メリット
- シンプルで分かりやすい
- 誰でも使える

### ❌ デメリット  
- メニューに載っていない料理は注文不可
- メニュー更新が手動
- ウェイターに質問できない

---

# 📖 MCP = スマートウェイター

## 使い方

1. ウェイターに話しかける
2. 「今日のメニューは？」と聞く
3. ウェイターが説明＆提案
4. 対話的に注文

---

# MCPの特徴

### ✅ メリット
- システムが「何ができるか」を説明
- AIと対話しながら使える
- メニュー変更に自動対応
- 質問に答えられる

---

# 📊 実際の使い方（1/2）

## タスク一覧の取得

### REST API
```
http://example.com/tasks にアクセス
→ タスク一覧表示
```

### MCP
```
AI: 「何ができる？」
→ システム: 「タスク一覧を見れます」
→ AI: 「見せて」
→ タスク一覧表示
```

---

# 📊 実際の使い方（2/2）

## 新規タスクの作成

### REST API
```
POST /tasks に JSONデータ送信
→ 事前にURLとフォーマットを知る必要がある
```

### MCP
```
AI: 「どんな操作ができる？」
→ システム: 「create_taskで作成できます」
→ AI: 「タスクを作成して」
→ 作成完了
```

# 💡 REST API が適している例

### ✅ ECサイトのAPI
- 商品検索、カート、注文
- 多くのユーザーが同じ方法で使用

### ✅ SNSのAPI
- 投稿、いいね、フォロー
- 広く使われる標準的な機能

---

# 💡 MCPが適している例

### ✅ AIアシスタント連携
- 「会議資料を準備して」
- AIが必要なツールを自動選択

### ✅ 社内業務の自動化
- 機能が頻繁に追加・変更
- 部署ごとに使い方が違う

### ✅ 複雑な業務フロー
- AIが複数のツールを組み合わせ

---

# 🤝 併用も可能

## 使い分け

```
外部向けAPI → REST API
     +
AI連携機能 → MCP
```

## 実際の例
- 一般ユーザー向け: REST API
- AIアシスタント向け: MCP
- **両方を提供可能**

---

# ✅ まとめ: 5つのポイント

---

# 1. 自己説明性

- **REST API:** ドキュメントを読む必要がある
- **MCP:** システムが「何ができるか」を自動で説明

---

# 2. AI対応

- **REST API:** AIに使わせるには追加設定が必要
- **MCP:** AIがそのまま理解して使える

---

# 3. 変更への対応

- **REST API:** 変更時にドキュメントも更新が必要
- **MCP:** システムが自動で最新情報を提供

---

# 4. 使い分け

- **REST API:** 一般的なWebサービス、外部公開API
- **MCP:** AI連携、社内ツール、エージェントシステム

---

# 5. 将来性

- **REST API:** 実績豊富で安定
- **MCP:** AI時代の新しい標準として期待
