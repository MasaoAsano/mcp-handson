# 🤔 一言で言うと

**REST API**  
レストランのメニュー（紙）

**MCP**  
スマートなウェイター（AI対応）

---

# 📖 REST API

## 紙のメニュー方式

1. メニューを見る
2. 注文を伝える
3. 料理が届く

---

# REST APIの特徴

**メリット**
- シンプル
- 誰でも使える

**デメリット**  
- メニュー外は注文不可
- 手動更新が必要
- 質問できない

---

# 📖 MCP

## スマートウェイター方式

1. ウェイターに話しかける
2. おすすめを聞く
3. 対話的に注文
4. 最適な料理が届く

---

# MCPの特徴

**メリット**
- 何ができるか自動説明
- AI対応
- 自動更新
- 質問に回答

---

# 📊 タスク一覧の取得

**REST API**
- URLにアクセス
- タスク一覧表示

**MCP**
- AI: 「何ができる？」
- システム: 「一覧を表示できます」
- タスク一覧表示

---

# 📊 タスクの作成（1/2）

**REST API**

- 専用URLにデータ送信
- 事前に仕様を確認が必要

---

# 📊 タスクの作成（2/2）

**MCP**

- AI: 「操作は？」
- システム: 「作成できます」
- AI: 「作成して」
- 完了
